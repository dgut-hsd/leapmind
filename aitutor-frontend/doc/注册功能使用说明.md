# 注册功能使用说明

## 概述

LoginPage2 现在支持完整的用户注册功能，用户可以在登录页面和注册页面之间自由切换。

## 功能特性

### ✅ 已实现功能

1. **登录/注册模式切换**
   - 点击"立即注册"切换到注册模式
   - 点击"立即登录"返回登录模式
   - 切换时自动清除错误提示

2. **注册表单验证**
   - 用户名：必填项
   - 邮箱：必填项，验证邮箱格式
   - 密码：必填项，最少6位
   - 确认密码：必填项，必须与密码一致
   - 真实姓名：可选
   - 手机号码：可选

3. **友好的用户体验**
   - 实时表单验证
   - 详细的错误提示
   - 成功提示
   - 注册成功后自动跳转到登录页面（2秒延迟）
   - 自动填充注册时使用的邮箱到登录表单

4. **美观的界面设计**
   - 现代紫色渐变背景
   - 毛玻璃效果卡片
   - 流畅的动画过渡
   - 响应式设计

## 表单字段说明

### 必填字段

| 字段 | 说明 | 验证规则 |
|------|------|----------|
| 用户名 | 用户登录使用的唯一标识 | 必填，不能为空 |
| 邮箱地址 | 用于接收通知和找回密码 | 必填，需符合邮箱格式 |
| 密码 | 账户安全密码 | 必填，至少6位 |
| 确认密码 | 确保密码输入正确 | 必填，必须与密码一致 |

### 可选字段

| 字段 | 说明 |
|------|------|
| 真实姓名 | 用户的真实姓名，用于个性化服务 |
| 手机号码 | 用于接收短信通知（可选） |

## 使用流程

### 1. 访问注册页面

用户访问应用时会看到登录页面，点击底部的 **"立即注册"** 链接切换到注册模式。

### 2. 填写注册信息

#### 必填信息
```
用户名: student001
邮箱地址: student001@example.com
密码: password123
确认密码: password123
```

#### 可选信息（推荐填写）
```
真实姓名: 张三
手机号码: 13800138000
```

### 3. 提交注册

点击 **"注册"** 按钮后：
- 系统会验证所有输入信息
- 如果有错误，会显示红色错误提示
- 如果成功，会显示绿色成功提示："注册成功！请登录"
- 2秒后自动切换到登录页面，邮箱地址会自动填充

### 4. 登录使用

注册成功后，使用刚才注册的邮箱（或用户名）和密码登录即可。

## 错误处理

### 常见错误提示

| 错误提示 | 原因 | 解决方法 |
|----------|------|----------|
| "请填写用户名、邮箱和密码" | 必填项未填写 | 检查并填写所有带 * 标记的必填项 |
| "请输入有效的邮箱地址" | 邮箱格式不正确 | 输入正确格式的邮箱，如 user@example.com |
| "密码长度至少为6位" | 密码太短 | 设置至少6位的密码 |
| "两次输入的密码不一致" | 密码和确认密码不匹配 | 确保两次输入的密码完全相同 |
| "用户名或邮箱已存在" | 该用户名或邮箱已被注册 | 更换用户名或使用其他邮箱 |
| "服务器错误，请稍后重试" | 后端服务异常 | 稍后重试或联系管理员 |
| "注册失败，请检查网络连接" | 网络问题 | 检查网络连接后重试 |

## 前端表单验证规则

```javascript
// 邮箱验证
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/

// 验证流程
1. 检查必填项（用户名、邮箱、密码）
2. 验证邮箱格式
3. 验证密码长度（>= 6）
4. 验证密码确认匹配
```

## API 调用

注册表单会调用后端的注册 API：

```javascript
POST /api/auth/register

// 请求体
{
  "username": "student001",
  "email": "student001@example.com",
  "password": "password123",
  "realName": "张三",      // 可选
  "phone": "13800138000"   // 可选
}

// 成功响应
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": 1,
    "username": "student001",
    "email": "student001@example.com",
    // ... 其他用户信息
  }
}
```
