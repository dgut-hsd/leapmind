# AI教育平台前端项目 - 关键交互流程分析

## 🎬 核心交互场景

该项目有两个核心交互场景：

1. **场景A：用户提问模式** - 学生在任何时刻可以提问
2. **场景B：PPT讲课模式** - 系统播放预制的讲课内容，学生可以打断提问

---

#### **入口点：TeacherPanel.jsx**

```javascript
// src/components/teacher/TeacherPanel.jsx (第16-76行)

const handleSendMessage = async (text) => {
  // 1. 显示用户消息
  setIsChatVisible(true);
  setMessages(prev => [...prev, 
    { sender: 'user', text }, 
    { sender: 'ai', isTyping: true }
  ]);

  try {
    // 2. 调用后端API
    const sessionId = state.currentSessionId || '';
    const result = await askQuestion(sessionId, text);
    const answer = result?.answer || '（无回答）';
    
    // 3. 更新聊天记录
    setMessages(prev => {
      const next = [...prev];
      const idx = next.findIndex(m => m.isTyping);
      if (idx !== -1) next[idx] = { sender: 'ai', text: answer };
      else next.push({ sender: 'ai', text: answer });
      return next;
    });

    // 4. 合成语音
    const tts = await synthesizeSpeech(sessionId, answer);
    if (tts) {
      const arrayBuf = await tts.arrayBuffer();
      const screenplay = {
        expression: 'neutral',
        talk: { message: answer }
      };
      // 5. 虚拟人物说话
      sharedViewer?.model?.speak(arrayBuf, screenplay);
    }
  } catch (err) {
    // 错误处理
    setMessages(prev => {
      const next = [...prev];
      const idx = next.findIndex(m => m.isTyping);
      if (idx !== -1) next[idx] = { sender: 'ai', text: '抱歉，回答失败' };
      else next.push({ sender: 'ai', text: '抱歉，回答失败' });
      return next;
    });
  }
};
```

#### **虚拟人物说话：model.speak()**

```javascript
// src/features/vrmViewer/model.js

async speak(arrayBuf, screenplay) {
  // 1. 解码音频
  const audioBuffer = await this._audioContext.decodeAudioData(arrayBuf);
  
  // 2. 触发表情和动作
  if (screenplay.expression) {
    this.emoteController?.playEmotion(screenplay.expression);
  }
  
  if (screenplay.talk?.message) {
    this.emoteController?.playHeadMotionByMessage(
      screenplay.talk.message,
      screenplay.expression
    );
  }
  
  // 3. 创建音频源并播放
  const source = this._audioContext.createBufferSource();
  source.buffer = audioBuffer;
  source.connect(this._lipSync.analyser);
  source.connect(this._audioContext.destination);
  
  // 4. 标记为正在说话（用于LipSync）
  this._lipSync.currentSource = source;
  source.start();
  
  // 5. 等待播放完成
  return new Promise(resolve => {
    source.onended = () => {
      this._lipSync.currentSource = null;
      resolve();
    };
  });
}
```

#### **LipSync实时分析**

```javascript
// src/features/lipSync/lipSync.js (update方法)

update() {
  // 1. 检查是否有活跃音源
  const active = !!this.currentSource || this.isMicrophoneConnected || mediaPlaying;
  
  // 2. 获取音频时域数据
  this.analyser.getFloatTimeDomainData(this.timeDomainData);
  
  // 3. 计算音量包络
  let peak = 0.0, rms = 0.0;
  for (let i = 0; i < TIME_DOMAIN_DATA_LENGTH; i++) {
    const sample = this.timeDomainData[i];
    peak = Math.max(peak, Math.abs(sample));
    rms += sample * sample;
  }
  rms = Math.sqrt(rms / TIME_DOMAIN_DATA_LENGTH);
  
  // 4. 非线性映射和平滑
  let env = 1 / (1 + Math.exp(-35 * (peak * 0.7 + rms * 0.3) + 3));
  if (env < 0.03) env = 0;
  env = Math.pow(env, 0.8) * 1.2;
  
  // 5. Attack/Release
  const attack = 0.35, release = 0.20;
  if (env > this._prevVolume) {
    this._prevVolume += (env - this._prevVolume) * attack;
  } else {
    this._prevVolume += (env - this._prevVolume) * release;
  }
  
  // 6. 计算频率数据
  this.analyser.getByteFrequencyData(this.frequencyData);
  
  // 7. 返回口型权重（复杂的频率分析）
  return {
    volume: this._prevVolume,
    weights: this._analyzeFrequency(),  // 返回口型权重
    active: active
  };
}
```

#### **头部动作触发**

```javascript
// src/features/emoteController/headMotionController.js

playMotionByContent(message, emotionOrStyle) {
  // 1. 优先使用情绪映射
  if (emotionOrStyle && EMOTION_TO_MOTION[emotionOrStyle]?.length) {
    this.playMotionByEmotion(emotionOrStyle);
    return;
  }
  
  // 2. 使用关键词匹配
  const type = this._detectMotionFromText(message);
  if (type) {
    this.playHeadMotion(type);
  }
}

_detectMotionFromText(text) {
  const order = ["bigShake", "bigNod", "lookUp", "tiltHead", "smallShake", "smallNod"];
  
  for (const type of order) {
    const list = this._triggers[type];
    if (list && list.some((kw) => text.includes(kw))) {
      return type;  // 找到匹配的关键词
    }
  }
  return null;
}

playHeadMotion(motionType) {
  const motionData = this._motionDatabase[motionType];
  if (!motionData || !this._headBone) return;
  
  // 1. 播放对应的表情
  const expression = MOTION_TO_EXPRESSION[motionType];
  if (this._expressionController && expression) {
    this._expressionController.playEmotion(expression);
  }
  
  // 2. 记录当前动作
  this._currentMotion = {
    type: motionType,
    startTime: this._clock.getElapsedTime(),
    duration: motionData.duration,
    keyframes: motionData.keyframes
  };
}
```

### B.2 语音中断处理详解

这是最复杂的流程，让我们仔细剖析：

#### **源码位置**

```javascript
// src/features/chat/voiceInterruptionHandler.js
// 在 src/components/teacher/TeacherPanel.jsx 中调用
```

#### **完整代码流程**

```javascript
// 1. 检测到用户语音中断时触发
if (state.isPlaying && !state.isInterrupted) {
  await handleVoiceInterruption(userText);
}

// 2. handleVoiceInterruption 函数
async function handleVoiceInterruption(userText) {
  // 2.1 标记中断状态
  state.isInterrupted = true;
  
  // 2.2 暂停当前PPT播放
  pausePlayback();
  
  // 2.3 显示UI提示 "正在处理你的问题..."
  updatePlaybackStatus();
  
  // 2.4 调用后端API获取回答
  try {
    const sessionId = state.currentSessionId || '';
    const result = await askQuestion(sessionId, userText);
    const answer = result?.answer || '我理解你的问题，但暂无更详细的回答。';
    
    // 2.5 合成语音
    const tts = await synthesizeSpeech(sessionId, answer);
    if (!tts) throw new Error('TTS synthesis failed');
    
    const arrayBuf = await tts.arrayBuffer();
    
    // 2.6 构建screenplay对象
    const screenplay = {
      expression: 'neutral',
      talk: { message: answer }
    };
    
    // 2.7 虚拟人物播放回答 (等待完成)
    await sharedViewer?.model?.speak(arrayBuf, screenplay);
    
    // 2.8 更新聊天面板（同步显示对话）
    setMessages(prev => [...prev,
      { sender: 'user', text: userText },
      { sender: 'ai', text: answer }
    ]);
    
  } catch (error) {
    console.error('语音中断处理失败:', error);
    sharedViewer?.model?.stopSpeaking();
  } finally {
    // 2.9 清除中断状态
    state.isInterrupted = false;
    
    // 2.10 继续播放PPT
    resumePlayback();
  }
}
```

## 💡 故障排查指南

### 虚拟人物不说话

```
检查清单:
1. [ ] 音频是否加载成功？
   → 检查synthesizeSpeech API返回
   
2. [ ] model.speak()是否被调用？
   → 在TeacherPanel.jsx中添加console.log
   
3. [ ] audioContext是否初始化？
   → 检查LipSync的AudioContext状态
   
4. [ ] 音频Blob是否被正确解码？
   → 在model.speak()中添加错误处理
```

### 口型不同步

```
检查清单:
1. [ ] LipSync是否在更新？
   → lipSync.update()是否在每帧调用
   
2. [ ] 音频源是否连接到analyser？
   → 检查audioSource.connect(analyser)
   
3. [ ] 权重值是否超过0-1范围？
   → 检查_analyzeFrequency()返回值
   
4. [ ] BlendShape权重是否应用到VRM？
   → 检查expressionController.lipSync()
```

### 头部动作不执行

```
检查清单:
1. [ ] 关键词是否在DEFAULT_TRIGGERS中？
   → 检查headMotionController.js
   
2. [ ] headBone是否正确获取？
   → vrm.humanoid.getNormalizedBoneNode(VRMHumanBoneName.Head)
   
3. [ ] 动作是否超时没有执行？
   → 检查currentTime是否超过duration
   
4. [ ] 多个动作是否冲突？
   → 确保_currentMotion只有一个活跃
```

### PPT播放中断处理失败

```
检查清单:
1. [ ] 语音识别是否初始化？
   → initializeSpeechRecognition()是否调用
   
2. [ ] 状态管理是否正确？
   → state.isInterrupted是否正确设置/清除
   
3. [ ] pausePlayback()是否生效？
   → 检查mainAudio.pause()是否调用
   
4. [ ] resumePlayback()是否正确恢复?
   → 确保currentSegmentIndex正确
```

## 📚 进阶主题

### 自定义表情-动作映射

你可以修改 `headMotionController.js` 中的映射：

```javascript
// 原始映射
const MOTION_TO_EXPRESSION = {
  bigNod: "happy",
  // ...
};

// 修改为：
const MOTION_TO_EXPRESSION = {
  bigNod: "surprised",  // 现在点头显示惊讶
  // ...
};
```

### 自定义关键词触发

```javascript
// 在TeacherPanel或其他地方调用：
const triggers = sharedViewer.model.emoteController.getHeadMotionTriggers();

// 修改并重新设置
triggers.bigNod.push("完美", "太棒了");
sharedViewer.model.emoteController.setHeadMotionTriggers("bigNod", triggers.bigNod);
```

    // 修改关键帧... 
  ]
}
