## 1. 项目概述

### 1.1 项目定位

对话数字人系统是一个集成3D虚拟人物渲染与智能对话交互的核心功能模块。系统通过先进的3D渲染技术展示虚拟形象，结合自然语言处理与语音合成技术，实现类人的对话交互体验。

### 1.2 核心价值

- **沉浸式教学体验**: 通过3D数字人形象增强互动性和视觉吸引力
- **自然对话交互**: 支持文本和语音两种输入方式，提供流畅的对话体验
- **智能表情动作**: 根据对话内容自动触发合适的面部表情和头部动作
- **实时口型同步**: 音频播放时实时分析并同步虚拟人物的嘴型变化

### 1.3 应用场景

- 用户提问答疑：用户输入问题，虚拟人物给出解答并配合相应的表情动作
- 语音交互：支持语音输入识别，降低输入门槛，提升交互便捷性

### 2.2 核心模块说明

#### 2.2.1 对话管理层
- **TeacherPanel.jsx**: React组件，负责对话UI的展示和用户交互处理
- **ChatInput.jsx**: 文本输入组件，支持键盘输入和语音输入
- **ChatDialog.jsx**: 对话消息展示组件，显示用户和AI的消息历史

#### 2.2.2 API服务层
- **pptApi.js**: 封装后端API调用
  - `askQuestion()`: 发送问题到后端，获取AI回答
  - `synthesizeSpeech()`: 将文本转换为语音音频

#### 2.2.3 3D渲染层
- **Viewer**: Three.js场景管理器，负责渲染循环
- **Model**: VRM模型管理器，整合表情、动作、口型等系统
- **LipSync**: 音频分析和口型同步系统
- **EmoteController**: 表情和头部动作控制器

## 3. 对话功能规格

### 3.1 输入方式

#### 3.1.1 文本输入

**功能描述**: 用户通过键盘在输入框中输入问题，按回车键或点击发送按钮提交

**技术实现**:
```javascript
// 位置: src/components/chat/ChatInput.jsx
- 输入框支持实时输入
- 支持 Enter 键提交
- 提交后清空输入框
- 自动聚焦输入框
```

**界面要求**:
- 输入框宽度自适应，带有圆角样式
- 发送按钮位于输入框右侧
- 输入框获得焦点时显示蓝色边框高亮
- 占位符文本："输入您的问题..."

#### 3.1.2 语音输入

**功能描述**: 用户点击语音按钮，通过麦克风进行语音输入，系统自动识别并转换为文本

**技术实现**:
```javascript
// 使用 Web Speech API (SpeechRecognition)
- 支持中文识别 (zh-CN)
- 实时显示识别中间结果
- 识别完成后自动提交
- 自动请求麦克风权限
```

**界面要求**:
- 语音按钮位于输入框左侧
- 录音时按钮显示高亮状态
- 录音时输入框显示"正在聆听..."
- 识别结果显示在输入框中（中间结果实时更新）

**错误处理**:
- 浏览器不支持语音识别时提示用户
- 麦克风权限被拒绝时显示提示
- 识别失败时恢复输入框状态

### 3.2 对话处理流程

#### 3.2.1 消息发送流程

```
用户输入/语音识别
    ↓
添加用户消息到聊天记录
    ↓
显示"AI正在思考..."占位符
    ↓
调用后端API (askQuestion)
    ↓
[成功] ←→ [失败]
    ↓              ↓
更新AI消息      显示错误消息
    ↓
语音合成 (TTS)
    ↓
虚拟人物播报 (speak)
    ↓
完成，等待下一个问题
```

#### 3.2.2 消息状态管理

系统维护以下消息状态：

- **用户消息**: `{ sender: 'user', text: string }`
- **AI思考中**: `{ sender: 'ai', isTyping: true }`
- **AI回答**: `{ sender: 'ai', text: string }`
- **错误消息**: `{ sender: 'ai', text: '抱歉，回答失败...' }`

#### 3.2.3 对话上下文

- 系统维护会话ID (`sessionId`)，用于保持对话上下文
- 每次提问都会携带历史对话上下文到后端
- 支持多轮对话，AI能够理解上下文关系

### 3.3 对话展示

#### 3.3.1 消息列表

**布局要求**:
- 消息列表支持滚动查看历史消息
- 用户消息右对齐，AI消息左对齐
- 消息气泡采用不同颜色区分（用户：蓝色，AI：灰色）
- 支持自动滚动到最新消息

**内容显示**:
- 文本消息支持换行显示
- 长文本自动换行，不溢出容器
- 支持消息时间戳显示（可选）

#### 3.3.2 输入状态指示

- 显示"正在思考..."动画（打字效果）
- 语音识别时显示"正在聆听..."提示
- 网络请求时显示加载状态

### 3.4 中断和恢复机制

当系统处于PPT讲课模式时：

**中断流程**:
1. 用户提问触发中断
2. 暂停PPT音频播放
3. 处理用户问题
4. 播放回答音频
5. 回答完成后恢复PPT播放

**实现要点**:
- 支持在任意时刻中断讲课
- 中断后准确恢复到中断位置
- 保持对话记录的完整性

## 4. 3D数字人功能规格

### 4.1 VRM模型管理

#### 4.1.1 模型加载

**支持格式**:
- VRM 1.0标准格式
- 基于GLTF/GLB格式
- 包含骨骼、表情、材质信息

**加载流程**:
```
1. 创建GLTFLoader
2. 注册VRMLoaderPlugin
3. 加载VRM文件
4. 初始化动画混合器
5. 创建表情和动作控制器
6. 调整相机位置适配模型
```

**模型要求**:
- 包含标准人形骨骼系统
- 支持BlendShape表情系统
- 支持lookAt注视系统
- 模型尺寸适中，适合教育场景

#### 4.1.2 模型渲染

**渲染参数**:
- 渲染分辨率：自适应容器大小
- 帧率：目标60fps
- 抗锯齿：启用
- 阴影：启用（提升真实感）

**相机设置**:
- 相机类型：透视相机 (PerspectiveCamera)
- 初始位置：自动适配模型（fitToModel）
- 相机距离：模型高度的0.9倍
- 垂直偏移：模型高度的16%

**灯光系统**:
- 环境光 (AmbientLight): 亮度0.6，提供基础照明
- 平行光 (DirectionalLight): 亮度0.9，模拟主光源
- 半球光 (HemisphereLight): 亮度0.6，增强立体感

### 4.2 表情系统

#### 4.2.1 表情类型

系统支持以下预设表情：

| 表情类型 | 标识符 | 视觉表现 | 触发场景 |
|---------|--------|---------|---------|
| 中性 | `neutral` | 平静面容 | 默认状态 |
| 开心 | `happy` | 微笑、眼眉上扬 | 积极回答、鼓励 |
| 悲伤 | `sad` | 嘴角下垂、眉毛低垂 | 抱歉、遗憾 |
| 愤怒 | `angry` | 眉头紧锁、嘴角下拉 | 强调、否定 |
| 惊讶 | `surprised` | 眼睛睁大、眉毛上扬 | 意外、惊喜 |

#### 4.2.2 表情控制

**控制方式**:
```javascript
// 通过EmoteController控制
emoteController.playEmotion('happy');  // 切换到开心表情
emoteController.playEmotion('neutral'); // 回到中性表情
```

**表情切换**:
- 表情之间支持平滑过渡
- 过渡时间约0.3-0.5秒
- 使用权重插值实现自然过渡

**表情触发逻辑**:
- 默认状态：`neutral`
- 回答问题开始时：根据回答内容情绪选择合适表情
- 回答问题结束时：逐渐恢复到`neutral`

### 4.3 头部动作系统

#### 4.3.1 动作类型

系统支持以下头部动作：

| 动作类型 | 标识符 | 动作描述 | 适用场景 |
|---------|--------|---------|---------|
| 大幅点头 | `bigNod` | 快速连续点头2-3次 | 肯定、完全同意 |
| 轻微点头 | `smallNod` | 缓慢点头1-2次 | 理解、认可 |
| 歪头 | `tiltHead` | 头部向一侧倾斜 | 思考、疑问 |
| 大幅摇头 | `bigShake` | 快速左右摇头 | 强烈否定 |
| 小幅摇头 | `smallShake` | 缓慢左右摇头 | 不认同、遗憾 |
| 抬头 | `lookUp` | 头部向上抬起 | 惊讶、回忆 |

#### 4.3.2 动作触发机制

**触发方式**:

1. **关键词触发**:
   ```javascript
   // 系统内置关键词库
   bigNod: ["完全同意", "非常赞同", "没错", "对极了"]
   smallNod: ["好的", "嗯", "明白了", "理解"]
   bigShake: ["不对", "错误", "不是这样", "不同意"]
   tiltHead: ["为什么", "怎么", "如何", "疑问"]
   ```

2. **情绪触发**:
   ```javascript
   // 根据情绪自动选择合适动作
   playHeadMotionByEmotion('happy')  // 可能触发smallNod
   playHeadMotionByEmotion('angry')  // 可能触发smallShake
   ```

3. **内容分析触发**:
   ```javascript
   // 分析消息内容，结合关键词和情绪
   playHeadMotionByMessage(message, emotion)
   ```

**优先级规则**:
1. 情绪匹配优先（如果情绪明确）
2. 关键词匹配其次
3. 随机选择默认动作

#### 4.3.3 动作执行

**动作时长**:
- 大幅动作：1.0-1.5秒
- 小幅动作：0.5-0.8秒
- 动作之间自动平滑过渡

**动作循环**:
- 每个动作支持循环播放（如连续点头）
- 循环次数根据动作类型自动调整

**动作叠加**:
- 支持表情与头部动作同时进行
- 动作开始时略微延迟（300ms），避免与表情切换冲突

### 4.4 口型同步系统

#### 4.4.1 口型分析

**音频分析技术**:
- 使用Web Audio API的AnalyserNode
- FFT大小：2048
- 平滑常数：0.7（降低抖动）

**分析流程**:
```
1. 获取音频时域数据 (getFloatTimeDomainData)
2. 计算音量包络 (RMS + Peak)
3. 获取音频频域数据 (getByteFrequencyData)
4. 计算频谱质心 (Spectral Centroid)
5. 映射到5个基本元音 (aa, ee, ih, oh, ou)
```

#### 4.4.2 口型类型

系统支持5种基本口型（Viseme）：

| 口型 | 标识符 | 对应音素 | 表现特征 |
|-----|--------|---------|---------|
| 张口大 | `aa` | a, o | 嘴巴张大 |
| 微笑 | `ee` | e, i | 嘴角横向拉伸 |
| 小开口 | `ih` | i | 嘴巴微微张开 |
| 圆唇 | `oh` | o | 嘴巴圆形 |
| 收唇 | `ou` | u | 嘴巴收缩 |

#### 4.4.3 实时同步

**同步频率**:
- 更新频率：60fps（每帧更新）
- 延迟：<50ms（实时响应）

**平滑处理**:
- Attack时间：0.35（上升快）
- Release时间：0.20（下降慢）
- 音量门限：0.03（低于此值不触发口型）

**权重计算**:
```javascript
// 根据频谱特征计算权重
weights = {
  aa: 0.0-1.0,  // 基于低频能量
  ee: 0.0-1.0,  // 基于高频能量
  ih: 0.0-1.0,  // 基于中高频
  oh: 0.0-1.0,  // 基于中频
  ou: 0.0-1.0   // 基于中频（略低）
}
```

### 4.5 待机动画系统

#### 4.5.1 空闲动画

**触发条件**:
- 用户无交互时间超过5秒
- 虚拟人物不在说话
- 系统处于空闲状态

**动画类型**:
- 呼吸动画：轻微的胸部起伏
- 眨眼动画：随机眨眼
- 头部微动：轻微的头部自然摆动

**动画管理**:
- 使用IdleAnimationManager统一管理
- 支持循环播放
- 用户交互时自动停止

#### 4.5.2 动画资源

- 支持VRM Animation格式 (.vrma)
- 动画文件：`idle_loop.vrma`
- 动画平滑循环，无明显跳跃

### 4.6 视线系统

#### 4.6.1 LookAt功能

**功能描述**:
- 虚拟人物的眼睛会跟随目标点
- 支持平滑注视过渡
- 目标点可以是鼠标位置或指定坐标

**实现方式**:
- 使用VRM的lookAt系统
- 平滑插件：VRMLookAtSmootherLoaderPlugin
- 注视角度限制：防止过度转动

#### 4.6.2 自动眨眼

**功能描述**:
- 虚拟人物自动进行眨眼动作
- 眨眼频率：每3-5秒一次
- 眨眼时长：0.1-0.2秒

## 5. 交互流程

### 5.1 标准对话流程

#### 流程时序图

```
用户                    前端界面              后端API              3D数字人
 │                        │                    │                    │
 │──输入问题─────────────▶│                    │                    │
 │                        │──askQuestion──────▶│                    │
 │                        │                    │──处理─────────────▶│
 │                        │                    │◀──返回答案─────────│
 │                        │◀──answer───────────│                    │
 │                        │                    │                    │
 │                        │──synthesizeSpeech─▶│                    │
 │                        │                    │──TTS──────────────▶│
 │                        │◀──audio────────────│                    │
 │                        │──speak(audio)──────┼───────────────────▶│
 │                        │                    │        [播放音频]   │
 │                        │                    │        [表情动作]   │
 │                        │                    │        [口型同步]   │
 │                        │                    │                    │
 │◀──显示回答─────────────│                    │                    │
```

#### 详细步骤

**步骤1: 用户输入**
- 用户在输入框中输入问题或点击语音按钮
- 系统验证输入内容不为空

**步骤2: 消息展示**
- 立即将用户消息添加到聊天记录
- 显示"AI正在思考..."占位符
- 打开对话面板（如果之前关闭）

**步骤3: API调用**
- 调用`askQuestion(sessionId, userText)`
- 等待后端返回回答（通常1-3秒）
- 如果超时或失败，显示错误消息

**步骤4: 语音合成**
- 调用`synthesizeSpeech(sessionId, answerText)`
- 获取音频ArrayBuffer
- 如果TTS失败，仅显示文本回答

**步骤5: 3D数字人播报**
- 调用`model.speak(arrayBuffer, screenplay)`
- 设置表情（根据回答内容）
- 延迟300ms后触发头部动作
- 播放音频并实时同步口型
- 音频播放完成后恢复正常状态

### 5.2 PPT讲课模式下的对话流程

#### 特殊处理

当系统处于PPT讲课模式时，对话流程需要特殊处理：

```
PPT讲课中
    ↓
用户提问（打断）
    ↓
暂停PPT音频播放
    ↓
[进入标准对话流程]
    ↓
对话完成
    ↓
恢复PPT音频播放（从暂停位置继续）
```

**关键实现**:
- `handleVoiceInterruption()`: 处理语音打断
- 保存PPT播放状态（当前片段、播放位置）
- 对话完成后调用`resumePlayback()`恢复播放

### 5.3 语音输入流程

#### 详细流程

```
用户点击语音按钮
    ↓
请求麦克风权限
    ↓
[权限被拒绝] ←→ [权限允许]
    ↓                    ↓
显示提示         创建SpeechRecognition
    ↓                    ↓
                   开始录音
                    ↓
                实时识别（显示中间结果）
                    ↓
                识别完成
                    ↓
                转换为文本
                    ↓
                自动提交（等同于文本输入）
```

**中间结果显示**:
- 识别过程中的临时文本会实时显示在输入框中
- 最终文本（isFinal）会自动替换中间文本
- 识别完成后立即提交

## 7. 接口定义

### 7.1 前端API接口

#### 7.1.1 askQuestion

**接口描述**: 向后端发送问题，获取AI回答

**调用方式**:
```javascript
const result = await askQuestion(sessionId, questionText);
```

**参数**:
- `sessionId` (string): 会话ID，用于维护对话上下文
- `questionText` (string): 用户输入的问题文本

**返回值**:
```javascript
{
  answer: string,      // AI回答的文本内容
  sessionId?: string   // 更新的会话ID（可选）
}
```

**错误处理**:
- 网络错误：返回`null`或抛出异常
- 超时处理：建议设置10秒超时
- 错误提示：显示"抱歉，回答失败，请稍后重试。"

#### 7.1.2 synthesizeSpeech

**接口描述**: 将文本转换为语音音频

**调用方式**:
```javascript
const audioResponse = await synthesizeSpeech(sessionId, text);
```

**参数**:
- `sessionId` (string): 会话ID
- `text` (string): 需要合成的文本

**返回值**:
- `Response`对象：包含音频数据的HTTP响应
- 使用`arrayBuffer()`方法获取音频二进制数据

**错误处理**:
- TTS失败时不阻塞对话流程，仅显示文本
- 静默失败，不显示错误提示

### 7.2 3D模型接口

#### 7.2.1 Model.speak()

**接口描述**: 让虚拟人物说话（播放音频并同步表情动作）

**调用方式**:
```javascript
await model.speak(audioBuffer, screenplay);
```

**参数**:
- `audioBuffer` (ArrayBuffer): 音频数据
- `screenplay` (Object): 剧本对象
  ```javascript
  {
    expression: string,     // 表情类型: 'happy', 'sad', 'neutral'等
    talk: {
      message: string       // 对话文本内容（用于关键词匹配）
    }
  }
  ```

**返回值**:
- `Promise<void>`: 音频播放完成后resolve

**行为**:
1. 停止当前待机动画
2. 设置表情
3. 延迟300ms后触发头部动作（根据消息内容和情绪）
4. 播放音频
5. 实时同步口型
6. 播放完成后恢复到待机状态

#### 7.2.2 Model.stopSpeaking()

**接口描述**: 停止当前语音播放

**调用方式**:
```javascript
model.stopSpeaking();
```

**使用场景**:
- PPT暂停时停止虚拟人物说话
- 用户打断时立即停止播放

#### 7.2.3 EmoteController接口

**表情控制**:
```javascript
emoteController.playEmotion(preset);  // preset: 'happy', 'sad', etc.
```

**头部动作控制**:
```javascript
emoteController.playHeadMotion(type);  // type: 'bigNod', 'smallNod', etc.
emoteController.playHeadMotionByMessage(message, emotion);
emoteController.playHeadMotionByEmotion(emotion);
```

**口型控制**:
```javascript
emoteController.lipSync(preset, value);      // 预设口型
emoteController.lipSyncWeights(weights);      // 精确权重控制
```

## 8. 性能要求

### 8.1 渲染性能

#### 帧率要求
- **目标帧率**: 60fps
- **最低帧率**: 30fps（可接受的最低体验）
- **帧率稳定性**: 帧时间波动<10%

#### 资源占用
- **内存占用**: VRM模型加载后<200MB
- **GPU占用**: 正常渲染<50%
- **CPU占用**: 主线程<30%（渲染循环）

### 8.2 响应性能

#### 交互响应
- **输入响应**: <100ms（用户输入到UI更新）
- **API响应**: 后端响应时间<3秒（目标<1秒）
- **TTS生成**: <2秒（目标<1秒）

#### 音频播放
- **音频延迟**: 从开始播放到口型同步<50ms
- **口型同步精度**: 与音频波形误差<100ms

### 8.3 加载性能

#### 模型加载
- **VRM加载**: <3秒（1MB模型）
- **动画加载**: <1秒
- **首屏渲染**: <5秒（包含模型和UI）

#### 优化策略
- 使用CDN加速资源加载
- 模型文件压缩（GLB格式）
- 懒加载非关键资源

## 9. 扩展性说明

### 9.1 表情扩展

#### 添加新表情

**步骤**:
1. 在`expressionController.js`中添加表情定义
2. 确保VRM模型包含对应的BlendShape
3. 在`emoteController.js`中注册新表情
4. 在表情触发逻辑中添加使用场景

**示例**:
```javascript
// 添加"困惑"表情
const CONFUSED_EXPRESSION = {
  name: 'confused',
  blendShapes: {
    brows_sad: 0.5,
    brow_trouble: 0.7
  }
};
```

### 9.2 动作扩展

#### 添加新头部动作

**步骤**:
1. 在`headMotionController.js`中定义动作关键帧
2. 添加动作到动作数据库
3. 设置触发关键词（可选）
4. 在动作选择逻辑中注册

**示例**:
```javascript
// 添加"左右看"动作
const lookAround = {
  duration: 1.5,
  keyframes: [
    { time: 0.0, rotation: { x: 0, y: 0, z: 0 } },
    { time: 0.5, rotation: { x: 0, y: -0.3, z: 0 } },
    { time: 1.0, rotation: { x: 0, y: 0.3, z: 0 } },
    { time: 1.5, rotation: { x: 0, y: 0, z: 0 } }
  ]
};
```

### 9.3 关键词扩展

#### 添加触发关键词

**位置**: `headMotionController.js` 或 `emoteConstants.js`

**方式**:
```javascript
const CUSTOM_TRIGGERS = {
  bigNod: [...DEFAULT_TRIGGERS.bigNod, "新关键词1", "新关键词2"]
};
```

### 9.4 模型替换

#### 替换VRM模型

**步骤**:
1. 准备新的VRM模型文件（.vrm）
2. 放置到`public/`目录
3. 在`viewer.js`的`loadVrm()`方法中修改URL
4. 确保新模型包含必要的BlendShape和骨骼

**注意事项**:
- 新模型需要包含所有支持的表情BlendShape
- 骨骼结构需要符合VRM标准
- 模型尺寸可能影响相机位置，需要调整`fitToModel()`参数

### 9.5 多语言支持

#### 扩展语音识别语言

**位置**: `ChatInput.jsx`的`handleVoiceInput()`

**修改**:
```javascript
const sr = createOneShotRecognition('en-US');  // 改为英语
```

#### 扩展TTS语言

**后端支持**: 需要后端TTS服务支持目标语言
- 当前支持：中文（zh-CN）
- 可扩展：英语、日语等

### 9.6 插件机制

#### 自定义动画插件

系统支持通过插件方式扩展动画：

// 创建自定义动画插件
class CustomAnimationPlugin {

    this.model = model;
  }

  }

### C. 故障排查

**常见问题**:
1. **虚拟人物不显示**: 检查VRM文件路径、控制台错误
