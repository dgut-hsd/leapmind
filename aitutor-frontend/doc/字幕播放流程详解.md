## 🎯 整体流程概览

从点击"播放"按钮到字幕显示，整个流程可以分为以下几个阶段：

```
用户点击播放按钮
    ↓
1. 获取音频和字幕数据（从后端API）
    ↓
2. 拆分音频文件（按分隔符切分）
    ↓
3. 启动播放器（开始播放第一个片段）
    ↓
4. 播放每个音频片段（循环播放）
    ↓
5. 发送字幕事件（通过浏览器事件系统）
    ↓
6. 接收并显示字幕（React组件更新）
```

export async function fetchMergedAudio(courseId, pageNumber) {
  const res = await fetch(
    `${VOICE_API_BASE}/api/speech/ppt/${courseId}/page/${pageNumber}/audio`,
    {
      credentials: 'include',

**说明**：

- 这是一个完整的音频文件，包含所有片段的音频
- 音频文件内部有特殊的分隔符来标记每个片段的边界

### 第四步：拆分音频文件

**音频拆分详解**：

**位置**：`src/features/chat/pptAudioSplit.js` 第4行

```javascript
export function splitAudioByDelimiter(arrayBuffer) {
  // 1. 定义分隔符：这是一个特殊的字节序列
  const delimiterPrefix = new Uint8Array([0xff, 0xfe, 0xfd, 0xfc]);
  const fullDelimiterLength = 8; // 4字节标记 + 4字节长度
  
  // 2. 将音频数据转换为字节数组
  const audioData = new Uint8Array(arrayBuffer);
  const segments = [];
  const delimiterPositions = [];
  
  // 3. 查找所有分隔符的位置
  for (let i = 0; i <= audioData.length - fullDelimiterLength; i++) {
    let found = true;
    for (let j = 0; j < delimiterPrefix.length; j++) {
      if (audioData[i + j] !== delimiterPrefix[j]) {
        found = false;
        break;
      }
    }
    if (found) {
      delimiterPositions.push(i);  // 记录分隔符位置
      i += fullDelimiterLength - 1;
    }
  }
  
  // 4. 根据分隔符位置切分音频
  let startIndex = 0;
  for (let i = 0; i < delimiterPositions.length; i++) {
    const endIndex = delimiterPositions[i];
    if (endIndex > startIndex) {
      const segmentData = audioData.slice(startIndex, endIndex);
      if (segmentData.length > 0) segments.push(segmentData);
    }
    startIndex = delimiterPositions[i] + fullDelimiterLength;
  }
  
  // 5. 过滤掉太小的片段（可能是噪音）
  const minSegmentSize = 1024;
  return segments.filter((seg) => seg.length >= minSegmentSize);
}
```

**通俗解释**：
- 想象音频文件是一根长绳子，分隔符就是绳子上的特殊标记
- 代码会找到所有标记的位置
- 然后在每个标记处"剪断"绳子，得到多个小段
- 每个小段就是一个音频片段

**为什么需要拆分？**
- 后端返回的是一个完整的音频文件
- 但我们需要按片段播放，每个片段对应一行字幕
- 拆分后可以精确控制每个片段的播放时机

### 第八步：播放当前片段

**位置**：`src/features/chat/pptController.js` 第80行

```javascript
export async function playCurrentSegment() {
  const currentIndex = state.currentSegmentIndex;
  
  // 1. 获取当前片段的数据
  const segment = state.audioSegments[currentIndex];
  const audioBlob = state.splitAudioBlobs[currentIndex];
  
  // 2. 发送字幕事件（关键！）
  try {
    const subtitleText = String(segment?.textContent || '').trim();
    if (subtitleText && typeof window !== 'undefined') {
      window.dispatchEvent(
        new CustomEvent('ppt-subtitle', 
          { detail: { text: subtitleText } }
        )
      );
    }
  } catch {}
  
  // 3. 播放音频（使用WebAudio API）
  const arrayBuf = await audioBlob.arrayBuffer();
  const screenplay = {
    expression: 'neutral',
    talk: { message: segment?.textContent || '' },
  };
  await sharedViewer?.model?.speak(arrayBuf, screenplay);
  
  // 4. 播放完成后，触发下一个片段
  onSegmentEnded();
}
```

**关键技术点**：

#### 8.1 字幕事件发送

```javascript
window.dispatchEvent(
  new CustomEvent('ppt-subtitle', 
    { detail: { text: subtitleText } }
  )
);
```

**说明**：
- `CustomEvent` 是浏览器提供的自定义事件API
- 事件名是 `'ppt-subtitle'`
- 字幕文本放在 `detail.text` 中
- 这个事件会被全局监听（任何组件都可以监听）

**为什么用事件而不是直接调用？**
- 播放器代码（`pptController.js`）不是React组件
- React组件（`SlideViewer.jsx`）需要接收字幕
- 事件系统实现了"解耦"：播放器不需要知道谁在监听

---

### 第九步：监听字幕事件

**位置**：`src/components/lecture/SlideViewer.jsx` 第241-260行

```javascript
useEffect(() => {
  const onSubtitle = (e) => {
    console.log('[调试]收到字幕事件：', e);
    const text = (e?.detail?.text || '').trim();
    console.log('[调试]提取字幕文本：', text);
    if (text) {
      console.log('[调试]更新字幕显示：', text);
      setRemoteSubtitleText(`<span class="subtitle-line">${text}</span>`);
    } else {
      console.log('[调试]未提取到字幕文本');
    }
  };
  
  // 注册事件监听器
  window.addEventListener('ppt-subtitle', onSubtitle);
  console.log('[调试]字幕事件监听器已注册');
  
  // 组件卸载时移除监听器（重要！避免内存泄漏）
  return () => {
    console.log('[调试]移除字幕事件监听器');
    window.removeEventListener('ppt-subtitle', onSubtitle);
  };
}, []);
```

**React Hook说明**：
- `useEffect` 是React的生命周期Hook
- 第二个参数 `[]` 表示只在组件挂载时执行一次
- `return` 函数会在组件卸载时执行（清理工作）

**事件监听流程**：
1. 组件挂载时，注册 `'ppt-subtitle'` 事件监听器
2. 当播放器发送事件时，`onSubtitle` 函数被调用
3. 从事件对象中提取字幕文本
4. 调用 `setRemoteSubtitleText` 更新React状态
5. React自动重新渲染组件，显示新字幕

### 第十一步：播放下一个片段

**位置**：`src/features/chat/pptController.js` 第143行

```javascript
export function onSegmentEnded() {
  console.log(`🔊 音频片段 ${state.currentSegmentIndex + 1} 播放结束`);
  
  // 防止重复处理
  if (onSegmentEndedProcessing) return;
  onSegmentEndedProcessing = true;
  
  // 检查是否还在播放
  if (!state.isPlaying || state.isInterrupted) {
    onSegmentEndedProcessing = false;
    return;
  }
  
  // 移动到下一个片段
  state.currentSegmentIndex++;
  
  // 检查是否所有片段都播放完了
  if (state.currentSegmentIndex >= state.audioSegments.length) {
    stopPlayback();
    onSegmentEndedProcessing = false;
    return;
  }
  
  // 延迟200ms后播放下一个片段（避免太快）
  setTimeout(() => {
    if (state.isPlaying && !state.isInterrupted) {
      playCurrentSegment();  // 递归调用，播放下一个片段
    }
    onSegmentEndedProcessing = false;
  }, 200);
}
```

**循环播放逻辑**：
1. 当前片段播放完成
2. 索引加1（`currentSegmentIndex++`）
3. 检查是否还有更多片段
4. 如果有，延迟200ms后播放下一个片段
5. 重复这个过程，直到所有片段播放完

## 🔧 关键技术点

### 1. 异步编程（Async/Await）

```javascript
const handleRemotePlay = async () => {
  const segments = await fetchSegments(...);  // 等待网络请求完成
  const audioBuffer = await fetchMergedAudio(...);  // 等待音频下载完成
  // 继续执行后续代码
};
```

**说明**：
- `async` 函数可以暂停执行，等待异步操作完成
- `await` 会等待Promise完成，然后继续执行
- 这样代码看起来像同步的，但实际上是异步的

### 2. 事件驱动架构

```
播放器代码（非React）
    ↓ 发送事件
window.dispatchEvent('ppt-subtitle')
    ↓
React组件监听
    ↓ 更新状态
setRemoteSubtitleText(...)
    ↓
组件重新渲染
    ↓
字幕显示更新
```

**优势**：
- 解耦：播放器和UI组件互不依赖
- 灵活：多个组件可以同时监听
- 可扩展：容易添加新功能

### 3. React状态管理

```javascript
const [remoteSubtitleText, setRemoteSubtitleText] = useState('初始文本');
```

**说明**：
- `useState` 创建状态变量和更新函数
- 调用 `setRemoteSubtitleText` 会触发组件重新渲染
- React会自动比较新旧值，只更新变化的部分

### 4. 二进制数据处理

```javascript
const audioData = new Uint8Array(arrayBuffer);  // 转换为字节数组
const segmentData = audioData.slice(startIndex, endIndex);  // 切片
const blob = new Blob([segmentData], { type: 'audio/wav' });  // 创建Blob
```

**说明**：
- `ArrayBuffer`：原始二进制数据
- `Uint8Array`：可以操作的字节数组
- `Blob`：浏览器可用的二进制对象

### 5. 全局状态管理

```javascript
// 在 pptState.js 中
export const state = { ... };

// 在任何地方都可以访问
import { state } from './pptState';
state.currentSegmentIndex = 0;
```

**说明**：
- 全局状态可以在任何模块中访问
- 不需要通过props层层传递
- 适合跨模块共享的数据

## 🎓 总结

整个字幕播放流程的核心是：

1. **数据获取**：从后端获取字幕文本和音频文件
2. **音频处理**：将完整音频文件拆分成多个片段
3. **事件通信**：使用浏览器事件系统连接播放器和UI
4. **状态管理**：使用React状态和全局状态管理数据
5. **循环播放**：逐个播放片段，同步显示字幕

**关键设计模式**：
- **事件驱动**：播放器和UI通过事件通信
- **状态管理**：React状态 + 全局状态
- **异步处理**：使用async/await处理网络请求
- **模块化**：每个功能独立成模块

希望这个解释能帮助你理解整个流程！如果有任何疑问，欢迎继续提问。
