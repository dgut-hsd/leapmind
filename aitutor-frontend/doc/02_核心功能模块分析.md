# AI教育平台前端项目 - 核心功能模块分析

## 🎯 模块概览

该项目分为4个核心模块，按优先级排列：

```
┌─────────────────────────────────────────────┐
│     1️⃣  VRM虚拟人物3D渲染系统              │
│         (Three.js + VRM模型)                │
└─────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────┐
│  2️⃣  虚拟人物表情和动作系统                 │
│  (EmoteController + HeadMotionController)   │
└─────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────┐
│     3️⃣  实时口型同步系统                    │
│        (LipSync + 音频分析)                 │
└─────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────┐
│    4️⃣  PPT讲课和问答系统                    │
│    (pptController + 语音识别)               │
└─────────────────────────────────────────────┘
```

## 📦 模块1：VRM虚拟人物3D渲染系统

### 1.1 架构图

```
Viewer (Three.js场景管理器)
  ├─ _scene (THREE.Scene)
  │   ├─ _camera (透视相机)
  │   ├─ _renderer (WebGL渲染器)
  │   └─ 灯光系统
  │       ├─ DirectionalLight (平行光)
  │       ├─ AmbientLight (环境光)
  │       └─ HemisphereLight (半球光)
  │
  ├─ _clock (时间管理)
  │
  └─ model (VRM模型实例)
      ├─ vrm (VRM对象)
      │   ├─ scene (VRM模型的场景图)
      │   ├─ humanoid (人形骨骼)
      │   └─ lookAt (注视目标)
      │
      ├─ mixer (THREE.AnimationMixer)
      │   └─ 管理VRM的所有动画
      │
      ├─ emoteController (表情和动作控制)
      │   ├─ ExpressionController
      │   └─ HeadMotionController
      │
      ├─ idleAnimationManager (空闲动画管理)
      │
      └─ lipSync (口型同步)
```

### 1.2 关键类和方法

#### **Viewer 类**

```javascript
class Viewer {
  constructor()
  // 初始化Three.js场景
  
  loadVrm(url)
  // 加载VRM模型
  
  setup(canvas)
  // 设置渲染目标canvas
  
  update(deltaTime)
  // 每帧更新渲染
}
```

**关键代码示例：**
- 初始化：创建Scene、Camera、Renderer、Light
- 加载VRM：使用GLTFLoader + VRMLoaderPlugin
- 渲染循环：requestAnimationFrame驱动update()

#### **Model 类**

```javascript
class Model {
  vrm                      // VRM对象
  mixer                    // 动画混合器
  emoteController          // 表情控制器
  idleAnimationManager     // 空闲动画管理
  
  async loadVRM(url)       // 加载VRM模型
  async loadAnimation(vrm) // 加载VRM动画
  async speak()            // 虚拟人物说话
  update(delta)            // 更新模型状态
}
```

### 1.3 渲染流程

```
1. 初始化阶段
   ├─ new Viewer() → 创建Three.js场景
   ├─ viewer.setup(canvas) → 绑定DOM
   └─ viewer.loadVrm(url) → 加载VRM模型

2. 每帧渲染循环
   ├─ requestAnimationFrame()
   ├─ viewer.update(deltaTime)
   │   ├─ model.update(delta)
   │   ├─ emoteController.update(delta)
   │   └─ lipSync.update()
   └─ renderer.render(scene, camera)

3. 用户交互
   ├─ 用户提问
   ├─ 调用 model.speak(audio, screenplay)
   └─ LipSync分析 + EmoteController播放动作
```

```
  │       ├─ 控制嘴形

  │
  └─ HeadMotionController (头部动作)
      ├─ playHeadMotion(type) 播放动作
      │   ├─ "bigNod" → 大幅点头 ✅ + 开心
      │   ├─ "smallNod" → 轻微点头 👍 + 放松
      │   ├─ "tiltHead" → 歪头 🤔 + 惊讶

      │   ├─ "smallShake" → 小幅摇头 👎 + 悲伤
      │   └─ "lookUp" → 抬头 😲 + 惊讶

          └─ "天哪" → lookUp
```

### 2.2 HeadMotionController 详解

这是项目最有趣的部分！让我们深入理解它：

#### **2.2.1 动作数据库**

每个动作都是一个 **关键帧动画**：

```javascript
bigNod: {
  duration: 1.2,  // 动作持续时间（秒）
  keyframes: [
    { time: 0.0, rotation: new THREE.Euler(0, 0, 0) },      // 起始
    { time: 0.3, rotation: new THREE.Euler(-0.4, 0, 0) },   // 低头
    { time: 0.6, rotation: new THREE.Euler(0.1, 0, 0) },    // 中途
    { time: 0.9, rotation: new THREE.Euler(-0.2, 0, 0) },   // 再次低头
    { time: 1.2, rotation: new THREE.Euler(0, 0, 0) }       // 回到起始
  ]
}
```

**Euler角的含义：**
- `x` - 前后旋转（点头）
- `y` - 左右旋转（摇头）
- `z` - 倾斜旋转（歪头）

#### **2.2.2 关键词触发映射**

```javascript
const DEFAULT_TRIGGERS = {
  bigNod: [
    "完全同意",
    "非常赞同",
    "没错",
    "对极了",
    // ...
  ],
  smallNod: [
    "好的",
    "嗯",
    "明白了",
    // ...
  ],
  // ...
}
```

**工作原理：**
```
AI说话 → 分析文本 → 匹配关键词 → 触发对应动作 + 表情
"完全同意" → 在DEFAULT_TRIGGERS中查找 → 匹配bigNod → 播放大幅点头+开心表情
```

#### **2.2.3 插值计算**

HeadMotionController使用 **三次Hermite插值** 实现平滑动作：

```javascript
_smoothstep(t) {
  return t * t * (3 - 2 * t);  // 平滑插值函数
}

// 在每一帧中：
// 1. 根据当前时间找到两个相邻的关键帧
// 2. 计算帧间的进度（0-1）
// 3. 应用平滑插值
// 4. 在两个旋转之间插值
// 5. 应用到head骨骼
```

### 2.3 表情和动作的搭配

```javascript
const MOTION_TO_EXPRESSION = {
  bigNod: "happy",       // 大幅点头 → 开心
  smallNod: "relaxed",   // 轻微点头 → 放松
  tiltHead: "surprised", // 歪头 → 惊讶
  bigShake: "angry",     // 大幅摇头 → 愤怒
  smallShake: "sad",     // 小幅摇头 → 悲伤
  lookUp: "surprised",   // 仰头 → 惊讶
};
```

当播放头部动作时，会自动播放对应的表情，使虚拟人物看起来更自然。

### 2.4 使用示例

```javascript
// 直接播放动作

// 根据情绪播放

// 根据内容自动判断
```

## 🔊 模块3：实时口型同步系统（LipSync）

### 3.1 系统架构

```
音频播放
  ↓
LipSync 实时分析
  ├─ FFT频谱分析
  │   └─ 获取频率信息
  │
  ├─ 时域波形分析
  │   ├─ 计算RMS（均方根）
  │   ├─ 计算Peak（峰值）
  │   └─ 非线性包络检测
  │
  └─ 口型权重计算
      └─ 10种口型的权重 (0-1)
          ├─ A, I, U, E, O（元音）
          └─ 其他辅音口型
  ↓
ExpressionController 应用口型
  └─ 更新VRM的嘴部BlendShape
```

### 3.2 LipSync工作流程

#### **3.2.1 音量包络计算**

```javascript
// 1. 获取时域数据
analyser.getFloatTimeDomainData(timeDomainData);

// 2. 计算音量
let peak = 0.0;      // 峰值
let rms = 0.0;        // 均方根

for (let i = 0; i < DATA_LENGTH; i++) {
  const sample = timeDomainData[i];
  peak = Math.max(peak, Math.abs(sample));
  rms += sample * sample;
}
rms = Math.sqrt(rms / DATA_LENGTH);

// 3. 组合音量 (70%峰值 + 30% RMS)
const combinedVolume = peak * 0.7 + rms * 0.3;

// 4. 非线性映射增强动态范围
let env = 1 / (1 + Math.exp(-35 * combinedVolume + 3));
if (env < 0.03) env = 0;  // 静音门限
env = Math.pow(env, 0.8) * 1.2;
env = Math.min(env, 1.0);
```

#### **3.2.2 Attack/Release 平滑**

```javascript
// 不同的上升/下降速度，使动作看起来更自然
const attack = 0.35;   // 音量上升快
const release = 0.20;  // 音量下降慢

if (env > this._prevVolume) {
  // 音量上升
  this._prevVolume = this._prevVolume + (env - this._prevVolume) * attack;
} else {
  // 音量下降
  this._prevVolume = this._prevVolume + (env - this._prevVolume) * release;
}
```

### 3.3 口型权重系统

```javascript
// LipSync返回的结果
{
  volume: 0.6,  // 0-1之间的音量
  weights: {
    A: 0.1,
    I: 0.3,
    U: 0.2,
    E: 0.1,
    O: 0.1,
    // ...其他口型
  },
  active: true  // 是否有活跃音源
}
```

这些权重直接映射到VRM模型的 **BlendShape**（混合形状），使嘴部随着语音自动变形。

### 3.4 音频源管理

```javascript

1. WebAudio节点

2. HTMLAudioElement
   └─ 用于播放音频文件

3. 麦克风输入
```

## 🎬 模块4：PPT讲课和问答系统

### 4.1 系统架构

```
PPT讲课系统
├─ pptState (全局状态管理)
│   ├─ isPlaying
│   ├─ currentSegmentIndex
│   ├─ audioSegments (音频分割)
│   └─ isInterrupted
│
├─ pptController (播放控制)
│   ├─ startPlayback()
│   ├─ pausePlayback()
│   ├─ stopPlayback()
│   ├─ playCurrentSegment()
│   └─ updatePlaybackControls()
│
├─ pptPlayback (播放逻辑)
│   ├─ 管理当前播放的音频片段
│   ├─ 更新字幕
│   └─ 触发虚拟人物动作
│
├─ pptVoiceControl (语音识别)
│   ├─ 初始化语音识别
│   ├─ 检测语音输入
│   └─ 触发语音中断处理
│
├─ pptApi (后端通信)
│   ├─ fetchSegments() 获取音频分割
│   ├─ askQuestion() 提问
│   └─ synthesizeSpeech() TTS合成
│
└─ voiceInterruptionHandler (语音中断处理)
    ├─ 暂停PPT播放
    ├─ 调用AI回答
    ├─ 虚拟人物说话
    └─ 恢复PPT播放
```

### 4.2 PPT讲课流程

```
用户点击"播放"
  ↓
1. 加载PPT音频数据
   ├─ fetchSegments() 获取分割数据
   └─ 生成音频Blob

2. 开始播放第一段
   ├─ playCurrentSegment()
   └─ 启动语音识别

3. 音频播放过程中
   ├─ 虚拟人物做动作（每段可能有的表情/动作数据）
   ├─ 字幕实时更新
   └─ LipSync同步口型

4. 用户说话（语音中断）
   ├─ 暂停PPT播放
   ├─ 检测用户语音
   └─ 触发 handleVoiceInterruption()

5. 处理用户问题
   ├─ 发送问题到后端
   ├─ 获取AI回答
   ├─ 合成语音(TTS)
   └─ 虚拟人物播放回答

6. 继续PPT播放
   └─ resumePlayback()

7. PPT播完
   ├─ 所有片段都播放完
   └─ 停止播放
```

### 4.3 关键数据结构

#### **pptState** - 全局状态

```javascript
const state = {
  // 播放状态
  isPlaying: false,        // 是否正在播放
  isPaused: false,         // 是否暂停
  isInterrupted: false,    // 是否被中断
  
  // 内容管理
  audioSegments: [],       // 音频分割数据数组
  splitAudioBlobs: [],     // 分割后的音频Blob
  currentSegmentIndex: 0,  // 当前播放片段索引
  
  // 会话管理
  currentSessionId: '',    // 当前会话ID
  
  // PPT数据
  slides: [],              // 幻灯片数据
  subtitles: {},           // 字幕映射
};
```

#### **音频分割对象**

```javascript
{
  start: 0,           // 起始时间（秒）
  end: 5.2,           // 结束时间（秒）
  text: "今天我们...", // 文本内容
  expression: "neutral", // 表情
  motion: null        // 动作（可选）
}
```

### 4.4 语音中断处理流程

```javascript
// 用户说话时触发
async function handleVoiceInterruption(userText) {
  // 1. 暂停PPT
  pausePlayback();
  
  // 2. 设置状态
  state.isInterrupted = true;
  
  // 3. 调用后端API获取回答
  const result = await askQuestion(state.currentSessionId, userText);
  const answer = result.answer;
  
  // 4. 合成语音
  const tts = await synthesizeSpeech(state.currentSessionId, answer);
  
  // 5. 虚拟人物播放回答
  const arrayBuf = await tts.arrayBuffer();
  const screenplay = {
    expression: 'neutral',
    talk: { message: answer }
  };
  sharedViewer.model.speak(arrayBuf, screenplay);
  
  // 6. 等待播放完成
  await new Promise(resolve => setTimeout(resolve, 2000));
  
  // 7. 恢复PPT播放
  resumePlayback();
}
```
