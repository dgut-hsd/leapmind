# 前端登录功能使用说明

## 概述

本项目已经实现了完整的用户登录和认证功能，基于后端提供的 JWT Token 认证方式。

## 文件结构

```
src/
├── services/
│   ├── api.js                 # API 基础服务（统一请求处理）
│   ├── authService.js         # 认证服务（登录、注册、用户信息）
│   ├── educationService.js    # 教育阶段服务
│   ├── courseService.js       # 课程服务
│   └── index.js              # 服务统一导出
├── utils/
│   └── tokenManager.js       # Token 管理工具
└── pages/
    ├── LoginPage.jsx         # 登录页面（样式1）
    └── LoginPage2.jsx        # 登录页面（样式2）
```

## 功能特性

### 1. 用户登录
- ✅ 支持用户名/邮箱 + 密码登录
- ✅ 自动保存 JWT Token 到 localStorage
- ✅ Token 过期自动检测
- ✅ 友好的错误提示

### 2. 用户认证
- ✅ 自动在请求头添加 Authorization Bearer Token
- ✅ Token 失效自动清除并跳转登录
- ✅ 页面刷新后保持登录状态

### 3. 用户信息管理
- ✅ 获取当前用户信息
- ✅ 更新用户信息
- ✅ 本地缓存用户信息

## 使用示例

### 1. 在组件中使用登录功能

```jsx
import React, { useState } from 'react';
import { login, ApiError } from '../services';

function LoginComponent() {
  const [username, setUsername] = useState('');
  const [password, setPassword] = useState('');

  const handleLogin = async () => {
    try {
      const result = await login(username, password);
      console.log('登录成功:', result.user);
      // 登录成功后的处理
    } catch (error) {
      if (error instanceof ApiError) {
        console.error('登录失败:', error.message);
      }
    }
  };

  return (
    <form onSubmit={(e) => { e.preventDefault(); handleLogin(); }}>
      <input 
        type="text" 
        value={username} 
        onChange={(e) => setUsername(e.target.value)} 
        placeholder="用户名或邮箱"
      />
      <input 
        type="password" 
        value={password} 
        onChange={(e) => setPassword(e.target.value)} 
        placeholder="密码"
      />
      <button type="submit">登录</button>
    </form>
  );
}
```

### 5. 检查登录状态

```jsx
import { hasValidToken, checkAuth } from '../services';

// 方式1：快速检查本地 token 是否存在且未过期
if (hasValidToken()) {
  console.log('本地有有效的 token');
}

// 方式2：向后端验证 token 是否真的有效
async function verifyAuth() {
  const isValid = await checkAuth();
  if (isValid) {
    console.log('用户已登录且 token 有效');
  } else {
    console.log('用户未登录或 token 已失效');
  }
}
```

## 错误处理

所有 API 调用都会抛出 `ApiError` 错误，包含以下信息：

```javascript
try {
  await login(username, password);
} catch (error) {
  if (error instanceof ApiError) {
    console.log('错误代码:', error.code);    // 200, 400, 401, 500 等
    console.log('错误消息:', error.message); // 具体错误信息
    console.log('错误数据:', error.data);    // 附加数据（可选）
  }
}
```

常见错误代码：
- `200`: 成功
- `400`: 请求参数错误
- `401`: 未授权（需要登录或 token 无效）
- `403`: 禁止访问（权限不足）
- `404`: 资源不存在
- `500`: 服务器内部错误

## 安全注意事项

1. **Token 存储**: Token 存储在 localStorage 中，请确保网站使用 HTTPS
2. **Token 过期**: Token 默认有效期为 1 小时（3600 秒），过期后需要重新登录
3. **自动清除**: Token 失效时会自动清除，避免使用无效 token
4. **密码安全**: 不要在前端存储用户密码

## 常见问题

### Q: 如何调试 API 请求？
A: 打开浏览器开发者工具的 Network 标签，查看具体的请求和响应。所有 API 请求都会在控制台输出错误日志。

### Q: Token 存在哪里？
A: Token 存储在浏览器的 localStorage 中，键名为 `auth_token`。

### Q: 如何强制用户重新登录？
A: 调用 `logout()` 函数清除 token，然后重定向到登录页面。

### Q: 为什么登录后刷新页面还是登录状态？
A: 因为 Token 存储在 localStorage 中，不会因为页面刷新而丢失。App.jsx 在启动时会检查 token 有效性。

### Q: 如何在请求中自动添加 Token？
A: 不需要手动添加，`api.js` 中的 `request` 函数会自动从 localStorage 读取 token 并添加到请求头。

## 开发建议

1. 使用 `services/index.js` 统一导入服务，保持代码整洁
2. 在组件中使用 try-catch 捕获错误，提供友好的用户提示
3. 根据 `ApiError.code` 提供不同的错误处理逻辑
4. 登录成功后，根据业务需求决定是刷新页面还是更新状态
