# 登录功能快速开始指南

## 1. 配置 API 地址

在项目根目录创建 `.env` 文件：

```env
VITE_API_BASE=http://localhost:8080
```

> 注意：如果后端运行在不同的端口或域名，请修改为实际地址。

## 2. 启动项目

```bash
# 安装依赖（首次运行）
npm install

# 启动开发服务器
npm run dev
```

## 3. 测试登录功能

访问 `http://localhost:5173`（或 Vite 提供的地址），将自动显示登录页面。

### 登录测试

使用测试账号登录：
- **用户名**: student001
- **密码**: password123

或使用任何在后端注册的账号。

## 4. 登录流程说明

```
用户输入账号密码
    ↓
调用 login() 函数
    ↓
发送 POST /api/auth/login
    ↓
接收 JWT Token
    ↓
保存 Token 到 localStorage
    ↓
保存用户信息到 localStorage
    ↓
更新应用状态（已登录）
    ↓
跳转到主页面
```

## 5. 主要文件说明

| 文件 | 作用 |
|------|------|
| `src/services/api.js` | 统一的 HTTP 请求处理，自动添加 Token |
| `src/services/authService.js` | 认证相关 API（登录、注册、用户信息） |
| `src/utils/tokenManager.js` | Token 存储和管理 |
| `src/pages/LoginPage2.jsx` | 登录页面（现代风格） |
| `src/App.jsx` | 应用入口，处理认证状态 |

## 6. 在组件中使用

### 示例1：简单登录

```jsx
import { login } from '../services';

async function handleLogin(username, password) {
  try {
    const result = await login(username, password);
    console.log('登录成功！用户信息:', result.user);
    // 刷新页面或更新状态
    window.location.reload();
  } catch (error) {
    alert('登录失败: ' + error.message);
  }
}
```

### 示例2：获取用户信息

```jsx
import { getUserProfile } from '../services';

async function loadUserInfo() {
  try {
    const user = await getUserProfile();
    console.log('当前用户:', user);
    return user;
  } catch (error) {
    console.error('获取用户信息失败:', error);
  }
}
```

### 示例3：用户登出

```jsx
import { logout } from '../services';

function handleLogout() {
  logout(); // 清除 token 和用户信息
  window.location.reload(); // 刷新页面
}
```

### 示例4：检查登录状态

```jsx
import { hasValidToken } from '../services';

function MyComponent() {
  const isLoggedIn = hasValidToken();
  
  return (
    <div>
      {isLoggedIn ? (
        <p>欢迎回来！</p>
      ) : (
        <p>请先登录</p>
      )}
    </div>
  );
}
```

## 8. 常见问题排查

### 问题1：登录后立即跳转到登录页

**原因**: 后端 API 地址配置错误或后端服务未启动

**解决**:
1. 检查 `.env` 文件中的 `VITE_API_BASE` 配置
2. 确认后端服务已启动
3. 检查浏览器控制台的网络请求错误

### 问题2：提示"未授权访问"

**原因**: Token 已过期或无效

**解决**:
1. 清除浏览器 localStorage
2. 重新登录获取新 Token

### 问题3：CORS 跨域错误

**原因**: 后端未配置 CORS

**解决**: 联系后端开发人员配置 CORS，允许前端域名访问

### 问题4：登录请求一直 pending

**原因**: 后端服务未启动或网络问题

**解决**:
1. 确认后端服务运行在正确的端口
2. 检查防火墙设置
3. 尝试在浏览器直接访问 API 地址

## 9. 调试技巧

### 查看 Token

在浏览器控制台输入：

```javascript
localStorage.getItem('auth_token')
```

### 查看用户信息

```javascript
JSON.parse(localStorage.getItem('user_info'))
```

### 清除登录状态

```javascript
localStorage.clear()
window.location.reload()
```

### 查看请求详情

打开浏览器开发者工具 → Network 标签 → 筛选 XHR/Fetch 请求
