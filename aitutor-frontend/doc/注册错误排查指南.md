# 注册错误排查指南

## 🔍 问题：注册时显示 "Internal Server Error"

当你看到 "Internal Server Error" 或 "服务器内部错误" 时，这通常表示后端服务器出现了问题。以下是详细的排查步骤。

## 📋 排查步骤

### 1️⃣ 检查浏览器控制台

打开浏览器开发者工具（F12），查看以下信息：

#### 查看 Network 标签
1. 打开开发者工具 → **Network** 标签
2. 尝试注册
3. 找到 `/api/auth/register` 请求
4. 点击查看详情：
   - **Status**: 应该是 500（服务器错误）
   - **Response**: 查看后端返回的具体错误信息
   - **Request Payload**: 查看发送的数据是否正确

#### 查看 Console 标签
现在代码已经添加了详细的调试日志，你应该能看到：
- `API 请求`: 显示发送的请求详情
- `API 错误响应`: 显示后端返回的错误详情
- `注册失败详情`: 显示完整的错误对象

### 2️⃣ 检查后端服务

#### 确认后端服务是否运行
```bash
# 检查后端服务是否启动
# 根据你的后端技术栈，使用相应的命令
```

#### 检查后端日志
查看后端服务器的日志，通常 500 错误会在日志中显示具体的错误原因，例如：
- 数据库连接失败
- 字段验证失败
- 代码异常
- 依赖服务不可用

### 3️⃣ 检查请求数据格式

根据 API 文档，注册请求应该发送以下数据：

```json
{
  "username": "string",
  "password": "string",
  "email": "string",
  "phone": "string",        // 可选
  "realName": "string",     // 可选
  "stageCode": "string",    // 可选
  "gradeCode": "string"     // 可选
}
```

**检查点**:
- ✅ 用户名、邮箱、密码是否都已填写
- ✅ 邮箱格式是否正确
- ✅ 密码长度是否至少 6 位
- ✅ 确认密码是否与密码一致

### 4️⃣ 检查 API 地址配置

确认 `.env` 文件中的 API 地址配置正确：

```env
VITE_API_BASE=http://localhost:8080
```

**检查点**:
- ✅ API 地址是否正确
- ✅ 后端服务是否运行在该地址
- ✅ 是否有 CORS 跨域问题（查看 Network 中的 CORS 错误）

### 5️⃣ 常见 500 错误原因

#### 原因 1: 数据库连接问题
**症状**: 后端日志显示数据库连接失败

**解决方法**:
- 检查数据库服务是否运行
- 检查数据库连接配置
- 检查数据库表是否存在

#### 原因 2: 字段验证失败
**症状**: 后端返回 "字段验证失败" 或类似信息

**解决方法**:
- 检查必填字段是否都已填写
- 检查字段格式是否符合要求
- 检查字段长度限制

#### 原因 3: 用户名或邮箱已存在
**症状**: 后端返回 "用户名已存在" 或 "邮箱已存在"

**解决方法**:
- 使用不同的用户名和邮箱
- 检查数据库中是否已有该用户

#### 原因 4: 后端代码异常
**症状**: 后端日志显示代码异常（如 NullPointerException）

**解决方法**:
- 查看后端日志的完整堆栈跟踪
- 联系后端开发人员修复

#### 原因 5: 依赖服务不可用
**症状**: 后端依赖的外部服务（如邮件服务）不可用

**解决方法**:
- 检查依赖服务是否运行
- 检查服务配置是否正确

## 📝 错误信息对照表

| 前端显示 | 后端状态码 | 可能原因 | 解决方法 |
|---------|-----------|---------|---------|
| "服务器内部错误" | 500 | 后端代码异常 | 查看后端日志，联系后端开发人员 |
| "请求参数错误" | 400 | 数据格式不正确 | 检查表单填写是否正确 |
| "用户名或邮箱已存在" | 400 | 重复注册 | 使用不同的用户名/邮箱 |
| "网络请求失败" | 0 | 网络问题或后端未启动 | 检查网络连接和后端服务 |
| "未授权访问" | 401 | Token 无效 | 重新登录 |

## 🔧 临时解决方案

如果问题暂时无法解决，可以：

1. **使用测试账号登录**
   - 如果已有测试账号，直接使用登录功能

2. **联系后端开发人员**
   - 提供完整的错误日志
   - 提供 Network 请求详情
   - 提供后端日志

3. **检查后端代码**
   - 如果是自己开发的后端，检查注册接口的实现
   - 确认数据库表结构是否正确
   - 确认字段验证逻辑是否正确
