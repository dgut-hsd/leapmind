# 需求文档

## 介绍

本功能旨在扩展AI老师系统，在文本润色完成后，将润色后的讲课内容按句号进行分割，然后逐句发送给阿里云TTS进行语音合成。系统将管理每个语音片段的播放状态，支持讲课被打断后从当前语音片段的开头重新开始播放，从而提供更好的用户体验和更精确的播放控制。

## 需求

### 需求 1

**用户故事：** 作为AI老师系统的用户，我希望系统能够将润色后的讲课内容按句子分割并逐句合成语音，以便实现更精细的播放控制。

#### 验收标准

1. 当系统接收到润色后的讲课文本时，系统应当按句号（。）进行文本分割
2. 当文本分割完成时，系统应当为每个句子分配唯一的序号标识
3. 当分割后的句子为空或只包含标点符号时，系统应当跳过该句子
4. 当分割完成时，系统应当按顺序逐句发送给阿里云TTS进行语音合成

### 需求 2

**用户故事：** 作为用户，我希望系统能够管理每个语音片段的播放状态，以便在播放被打断时能够准确恢复。

#### 验收标准

1. 当每个句子的TTS合成完成时，系统应当存储该语音片段及其对应的文本内容
2. 当语音片段开始播放时，系统应当记录当前播放的片段序号
3. 当播放被打断时，系统应当保存当前播放位置的片段序号
4. 当用户请求继续播放时，系统应当从保存的片段序号开始重新播放

### 需求 3

**用户故事：** 作为用户，我希望在讲课被打断后重新开始时，能够从当前语音片段的开头继续播放，而不是从整个讲课的开头重新开始。

#### 验收标准

1. 当播放被中断时，系统应当记录当前正在播放的语音片段序号
2. 当用户请求继续播放时，系统应当从记录的片段序号开始播放
3. 当从中断位置继续播放时，系统应当从该片段的开头开始，而不是中间位置
4. 当继续播放时，系统应当按顺序播放后续的所有语音片段

### 需求 4

**用户故事：** 作为开发人员，我希望系统能够高效地管理多个语音片段，避免内存溢出和性能问题。

#### 验收标准

1. 当语音片段数量较多时，系统应当实现流式处理，避免同时加载所有片段到内存
2. 当语音片段播放完成时，系统应当及时释放该片段占用的内存资源
3. 当TTS合成失败时，系统应当记录失败的片段并提供重试机制
4. 当系统检测到内存使用过高时，系统应当限制同时处理的语音片段数量

### 需求 5

**用户故事：** 作为用户，我希望系统能够提供播放进度信息，让我了解当前播放状态和剩余内容。

#### 验收标准

1. 当开始播放时，系统应当提供总片段数量和当前播放片段序号
2. 当片段播放完成时，系统应当更新播放进度信息
3. 当播放暂停或中断时，系统应当保持最后的进度状态
4. 当播放完成时，系统应当重置播放状态为初始状态

### 需求 6

**用户故事：** 作为系统管理员，我希望能够配置文本分割和语音片段管理的相关参数，以适应不同的使用场景。

#### 验收标准

1. 当系统启动时，系统应当加载分句分割符的配置（默认为句号）
2. 当配置最大片段长度时，系统应当在分句后检查长度并进行二次分割
3. 当配置最大同时处理片段数时，系统应当限制并发TTS请求数量
4. 当配置播放缓冲区大小时，系统应当预先合成指定数量的后续片段
