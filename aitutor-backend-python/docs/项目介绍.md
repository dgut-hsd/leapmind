## 项目介绍（LandPPT）

LandPPT 是一个基于大语言模型（LLM）的智能演示文稿生成平台。它将“需求→大纲→内容与设计→导出”的完整流程自动化，并提供可视化 Web 界面、模板系统、图像服务与研究系统，支持 OpenAI 兼容接口与本地模型（Ollama）。

- **核心能力**
  - **一键生成**：从主题与场景到完整 PPT（HTML/PDF/PPTX）
  - **多 AI 提供商**：OpenAI、Anthropic、Google Gemini、Ollama 等
  - **研究增强**：Tavily、SearXNG 搜索与网页内容抽取、结构化分析
  - **图像智能**：本地/网络/AI 生成三合一，智能匹配与缓存
  - **模板系统**：全局母版模板与多样化场景模板、AI 生成模板

## 系统架构总览

- **Web/API 层**（FastAPI）
  - 应用入口：`src/landppt/main.py` 创建 FastAPI 应用、注册中间件与路由，挂载静态目录与缓存目录。
  - Web 界面路由：`src/landppt/web/routes.py` 提供仪表盘、项目管理、编辑、导出等页面。
  - API 路由：
    - `api/landppt_api.py`：项目创建、生成流程、导出等核心接口
    - `api/openai_compat.py`：OpenAI 兼容接口（`/v1`）
    - `api/global_master_template_api.py`：全局母版模板管理
    - `api/config_api.py`：系统/AI 配置管理
    - `api/image_api.py`：图像服务状态、生成/搜索等
    - `api/database_api.py`：数据库健康、迁移、初始化等
- **认证与安全**
  - 中间件：`auth/middleware.py`（路由豁免、鉴权跳转等）
  - 登录/会话：`auth/routes.py`、`auth_service.py`
- **业务服务层**
  - PPT 生成：`services/enhanced_ppt_service.py`、`services/ppt_service.py`
  - 图像管线：`services/image/…`、`services/ppt_image_processor.py`
  - 研究系统：`services/research/…`、`services/deep_research_service.py`
  - 模板管理：`services/global_master_template_service.py`
  - 项目/流程：`services/project_manager.py`、`services/db_project_manager.py`
- **AI 抽象层**
  - 抽象与实现：`ai/base.py`、`ai/providers.py`
  - 统一选择工厂与配置：`ai/providers.py`、`core/config.py`
- **数据访问层**
  - 模型：`database/models.py`
  - 仓储：`database/repositories.py`
  - 服务：`database/service.py`
  - 初始化与会话：`database/database.py`、`database/migrations.py`
- **配置系统**
  - `core/config.py` 使用 Pydantic Settings 从 `.env` 加载，集中管理 AI/研究/图像/应用参数。

## 启动流程

1. `python run.py` → 导入 `landppt.main`，创建 FastAPI 应用。
2. 加载中间件（CORS、认证）、注册路由（Web、API、OpenAI 兼容等）。
3. 启动事件：`init_db()` 初始化数据库，`ensure_default_templates_exist_first_time()` 保证默认模板存在。
4. 挂载静态目录 `…/web/static` 与临时目录 `/temp`（图像缓存与导出过程临时文件）。
5. 根路由 `"/"` 跳转至仪表盘，健康检查 `"/health"` 可用于存活探测。

## 核心业务流程（从主题到可导出的 PPT）

### 1) 需求输入与项目创建

- 入口：
  - Web 界面：在“新建项目”页面填写主题、受众、页数范围、场景模板等参数。
  - API：`POST /api/projects` 或 `POST /api/outline/generate`（参见 `api/landppt_api.py`）。
- 存储：通过 `DatabaseService` 创建 `Project` 记录与对应的 TODO Board、版本、初始幻灯数据等。

### 2) 大纲生成（AI 驱动）

- 服务：`EnhancedPPTService.generate_outline` 结合主题、场景与提示词模板（`services/prompts/*`），调用统一 AI 提供商接口（`ai/providers.py`）。
- 原理：
  - 使用系统提示与场景提示，约束结构、层级与页数。
  - 产出结构化大纲（章节、要点、每页主题与内容提示）。

### 3) 幻灯片 HTML 生成（模板参考 + 创意生成）

- 服务：`EnhancedPPTService.generate_slides_from_outline`。
- 模板系统：
  - 全局母版模板（`GlobalMasterTemplate`）由 `global_master_template_service.py` 管理，可在 Web 界面选择/组合模板。
  - 生成时读取模板 HTML 作为“风格参考”，AI 生成“风格一致但内容创新”的每页 HTML（占位符如 `{{ page_title }}`、`{{ page_content }}` 等）。
- 结果：为每页生成完整、可渲染的 HTML 片段，遵循 16:9（1280×720）规范，CSS 内联保证导出一致性。

### 4) 图片处理（本地/网络/AI 生成）

- 处理器：`services/ppt_image_processor.py` 根据单页内容与需求，决定图片来源比例与数量。
- 图像服务：`services/image/image_service.py`
  - 提供者抽象：搜索（Unsplash、Pixabay、SearXNG/Google Images）、AI 生成（DALL·E、SiliconFlow、Stable Diffusion、Pollinations）、本地图库与用户上传。
  - 匹配与评分：`ImageMatcher` 根据语义与版式匹配度选优。
  - 缓存：`ImageCacheManager` 将生成/下载的图片进行本地缓存，构造可外链访问的绝对 URL（通过 `/temp` 与 `url_service`）。

### 5) 研究增强（可选）

- 服务：`services/research/enhanced_research_service.py` 与 `deep_research_service.py`。
- 流程：
  - 搜索引擎：Tavily 与 SearXNG 并行检索，聚合去重。
  - 网页抽取：抽取正文、标题、时间等结构化内容；可按 `max_tokens` 分块。
  - 分析归纳：通过统一 AI 提供商做要点提炼与事实性增强，产出“增强大纲/参考素材”。
- 集成：研究结果可用于强化大纲与正文生成、提升时效性与事实覆盖。

### 6) 导出（HTML/PDF/PPTX）

- HTML：`web/routes.py` 打包每页 HTML 到 ZIP；可单页/整站导出。
- PDF：基于 `pyppeteer` 或 `playwright` 的无头浏览器渲染，逐页合并为 PDF（保证分页与分辨率一致）。
- PPTX：调用 Apryse SDK（需设置 `APRYSE_LICENSE_KEY`），将 HTML/图片/文本结构转为 PPTX。

## AI 提供商抽象与调用原理

- 统一协议：`ai/base.py` 定义 `AIProvider`、`AIMessage`、`AIResponse` 等抽象。
- 实现：`ai/providers.py` 提供 `OpenAIProvider`、`AnthropicProvider`、`GoogleProvider`、`OllamaProvider` 等。
- 选择：`AIProviderFactory.create_provider()` 根据 `core/config.py` 中的默认提供商或用户设置，生成实例。
- 用法：业务服务仅面向抽象接口发起 `chat_completion` / `stream_chat_completion`，实现层负责各家 API 细节与重试、限流、超时等处理。

---

## 研究系统原理

- 方法论：DEEP（Define/Explore/Evaluate/Present）四步法；
- 提供者：
  - Tavily：结构化搜索结果（标题、URL、摘要、可信度、时间）。
  - SearXNG：多引擎聚合，提供更广覆盖的结果与去重能力。
- 内容抽取与分块：按配置进行网页正文抽取与按 `max_tokens` 分块，减少上下文丢失。
- AI 分析：将检索与抽取结果交由 AI 总结与归纳，形成“研究报告/增强大纲”。

---

## 图像服务原理

- 统一模型：`ImageService` 聚合“搜索/生成/上传/本地存储”等能力，暴露查找与生成接口。
- 提供者注册：`services/image/providers/base.py` 与各具体实现（DALL·E、SiliconFlow、Pollinations、Unsplash、Pixabay 等）。
- 匹配：根据幻灯片语义、占位与尺寸，智能选择与裁剪；支持统一尺寸策略保证版式一致。
- 缓存：生成或下载后的图片缓存到本地，并通过 `/temp` 暴露静态访问路径，便于前端预览与导出。

---

## 模板系统与页面生成

- 数据结构：`GlobalMasterTemplate` 作为全局母版（包含完整 HTML 与内联 CSS），支持标签、检索与启用状态。
- 生成逻辑：以模板 HTML 为“风格参考”，AI 生成“内容与结构”并保持风格一致；占位符统一，导出时可替换填充。
- API：`api/global_master_template_api.py` 支持创建、查询、启用、AI 生成模板等。

---

## 数据库模型与存储

- 主要实体（见 `database/models.py`）：
  - `User`、`UserSession`：用户与会话
  - `Project`：项目主体（主题、受众、场景、状态、页数、时间戳）
  - `SlideData`：每页数据（标题、要点、HTML、图片引用、AI 元数据）
  - `TodoBoard` / `TodoStage`：流程看板与节点（需求确认→大纲→生成→校对→导出）
  - `ProjectVersion`：版本与回溯
  - `PPTTemplate`、`GlobalMasterTemplate`：模板体系
- 访问模式：Repository（`database/repositories.py`）+ Service（`database/service.py`）
- 默认数据库：SQLite，亦可扩展至其他数据库（通过 SQLAlchemy 配置）。

## 配置与环境变量（关键项）

配置集中在 `core/config.py`（Pydantic Settings），由 `.env` 提供：

- **AI 提供商**：`DEFAULT_AI_PROVIDER`、`OPENAI_API_KEY`、`ANTHROPIC_API_KEY`、`GOOGLE_API_KEY`、`ENABLE_LOCAL_MODELS` 等
- **研究**：`TAVILY_API_KEY`、`SEARXNG_HOST`、`RESEARCH_PROVIDER`、`RESEARCH_ENABLE_CONTENT_EXTRACTION` 等
- **图像**：`PIXABAY_API_KEY`、`UNSPLASH_ACCESS_KEY`、`SILICONFLOW_API_KEY`、`ENABLE_IMAGE_SERVICE` 等
- **导出**：`APRYSE_LICENSE_KEY`（导出 PPTX）
- **应用**：`HOST`、`PORT`、`BASE_URL`、`LOG_LEVEL`、`ENABLE_STREAMING` 等

## 开发与扩展建议

- **新增 AI 提供商**：在 `ai/providers.py` 中实现并注册；在配置中增加参数，服务层无需改动。
- **新增研究提供商**：在 `services/research/` 下实现 Provider 与抽取器，接入增强研究服务。
- **新增图像提供商**：实现 `ImageSearchProvider` / `ImageGenerationProvider` 并注册到 `provider_registry`。
- **自定义模板**：通过 `global_master_template_api` 导入或 AI 生成，前端页面中选择应用。
- **流程定制**：在 `EnhancedPPTService` 中插入钩子（如图片策略、页面结构、提示词）以适配行业场景。

## 常见问题（运维/使用）

- 端口占用：`PORT` 改为其他端口，或释放占用 `taskkill /PID <PID> /F`。
- Playwright/Chromium 缺失：执行 `python -m playwright install chromium`。
- Apryse 安装：安装时增加 `--extra-index-url https://pypi.apryse.com` 并配置 `APRYSE_LICENSE_KEY`。
- Torch 与 CUDA：Windows 建议先用 CPU 版；GPU 需与本机 CUDA 版本匹配的索引源。

## 快速上手（摘要）

1) 创建环境（建议 Python 3.11）：

```bash
pip install -e . --extra-index-url https://pypi.apryse.com
python -m playwright install chromium
```

2) 配置 `.env`（最少项）：

```
HOST=0.0.0.0
PORT=8000
SECRET_KEY=change-me
# 可选 OPENAI_API_KEY / APRYSE_LICENSE_KEY 等
```

3) 启动：

```bash
python run.py
# Web: http://localhost:8000  Docs: http://localhost:8000/docs
```
